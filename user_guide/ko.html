코리더로 무엇을 할 수 있나요?
단촐한 외형에도 불구하고 KOReader는 매우 강력하고 사용자 정의가 가능한 응용 프로그램입니다. 아래 목록을 확인하여 코리더로 무엇을 할 수 있는지 알아보세요. 이 모든 기능은 이 안내서에 설명되어 있습니다.

  • 적합한 계정이 있는 (Dropbox/FTP/Webdav) 클라우드 저장소
  • KOReader는 SSH 서버 역할을 할 수 있으므로, SFTP 프로그램이나 파일 관리자를 사용하여 파일을 직접 전송할 수 있습니다
  • Calibre 플러그인을 사용하면, Calibre 라이브러리의 책을 Wi-Fi를 통해 코리더 기기로 바로 보낼 수 있습니다
  • 뉴스 다운로더 플러그인을 사용하면 RSS와 Atom 뉴스 항목을 HTML 파일로 장치에 다운로드할 수 있습니다
  • Wallabag 플러그인을 사용하면 Wallabag 계정(Pocket과 같은 서버 기반 나중에 읽기 서비스)에서 글을 검색할 수 있습니다

사용자 인터페이스
KOReader는 그라데이션과 다양한 회색 음영으로 인해 전자 잉크 화면에서 깜박임과 고스트가 발생하여 읽기 환경이 저하될 수 있으므로, 흑백 사용자 인터페이스 위주로 단순합니다.

KOReader에는 두 개의 주요 인터페이스가 있습니다: 읽기 화면과 파일 탐색기.

읽기 화면
읽기 화면에는 두 개의 도구 모음이 있습니다. 상단 메뉴에는 KOReader의 일반적인 사용과 관련된 메뉴 항목이 있습니다. 여기에서 책, KOReader 환경 설정, 플러그인 등에 접근할 수 있습니다. 하단 메뉴는 글꼴 굵기/크기, 줄 간격 등과 같은 문서 서식 지정 옵션에 중점을 둡니다. 텍스트 모양을 조정하는 동안 주로 하단 메뉴와 상호 작용하게 됩니다.

오른쪽에 KOReader의 기본 탭 영역이 표시됩니다. 상단 메뉴 또는 하단 메뉴를 표시하려면 표시된 영역을 탭하면 됩니다. 위쪽 영역을 아래로 스와이프하여 상단 메뉴를 표시할 수도 있습니다. Kindle 4와 같은 비터치 장치에서는 엔터/확인 동작에 해당하는 버튼을 눌러 하단 메뉴를 열 수 있습니다.

이 메뉴를 통해 상단 메뉴와 하단 메뉴를 동시에 열도록 상단 메뉴 영역을 설정할 수 있습니다:

 상단 메뉴 → → 탭과 제스처 → 메뉴 활성화하기
모서리 탭 영역은 탭, 더블 탭, 두 손가락 탭, 탭하고 누르기 등 다양한 유형의 제스처에 사용할 수 있습니다.

이전 페이지와 다음 페이지 영역은 동시에 두 번 탭 제스처에 사용할 수도 있습니다.

하나의 항목만 표시되는 경우 상태 표시줄 영역을 탭하여 상태 표시줄 항목 사이를 이동할 수 있습니다. 모든 항목이 표시되는 경우 탭하면 상태 표시줄이 표시되거나 숨겨집니다. 자세한 내용은 "상태 표시줄을 구성하려면 어떻게 해야 하나요?" 제목을 확인하세요.

파일 탐색기
KOReader를 처음 실행하면 파일 탐색기가 표시됩니다. 읽기 화면에서 다음과 같이 액세스할 수 있습니다:
 상단 메뉴 →
기본적으로 파일 탐색기에는 기본 라이브러리 폴더인 홈이 표시되며, 폴더를 길게 탭하여 변경할 수 있습니다. 화면 상단에서 아래로 스와이프하여 상단 메뉴를 통해 책이 표시되거나 정렬되는 방식 등 다양한 옵션을 변경할 수 있습니다.

이미지에서 동그라미로 표시된 항목은 아래에서 설명합니다:

재설정: 이 옵션은 기본적으로 구성 파일을 삭제하여 책을 초기화합니다.

 이 옵션을 선택하면 이 책에 있는 모든 메모와 강조 표시를 잃어버리게 됩니다.

다음 프로그램으로 열기: 코리더에는 다양한 파일 유형을 표시하기 위한 다양한 엔진이 있습니다. 경우에 따라 다른 엔진에서 파일을 열어 다른 기능을 사용할 수 있습니다. 바로 아래의 아카이브 파일 보기와 이미지 파일 보기 섹션을 참조하세요.

표지 무시: 어떤 이유로 이 책의 표지를 사용하지 않으려는 경우(표지가 잘못되었거나 전자 잉크 화면에서 표지가 잘 보이지 않을 수 있음) 이 옵션을 선택할 수 있습니다.

메타데이터 무시: 책 메타데이터가 올바르지 않으면 이름 필드에 쓸모없는 정보가 표시될 수 있습니다. 메타데이터 대신 파일 이름을 표시하려면 이 옵션을 선택합니다.

캐시된 책 정보 새로 고침: 코리더는 책을 라이브러리에 추가할 때 책의 메타데이터와 책 표지를 캐시합니다. 장치에 복사한 후 책을 일부 변경하는 경우 이 정보를 업데이트해야 할 수 있습니다. 잘못된 표지나 메타데이터가 표시되는 경우 이 메뉴 항목을 사용하여 책 정보를 새로 고칠 수 있습니다.

내 책에 접근하기
다음과 같은 방법으로 책과 기사에 접근할 수 있습니다:
  • 파일 탐색기
  • 즐겨찾기
  • 내력

이러한 각 대화 상자에 제스처를 지정할 수 있습니다. 예를 들어 책을 읽는 동안에는 제스처를 사용해 내력이나 즐겨찾기에 접근하고 파일 탐색기를 거치지 않고 다른 책을 열 수 있습니다.

또한 다음을 통해 즐겨찾기나 내력 화면에서 KOReader가 시작되도록 설정할 수도 있습니다:

 상단 메뉴(파일 탐색기에서) → → 다음으로 시작합니다
사용자 인터페이스 팁

스크롤 유형
코리더에는 세 가지 유형의 터치 스크롤이 있습니다. 다음에서 접근할 수 있습니다:
 상단 메뉴 → → 탭과 제스처 → 스크롤
 스크롤을 사용하려면 하단 메뉴에서 연속 모드를 활성화해야 합니다.

클래식 스크롤: 이 모드는 휴대폰/태블릿 스크롤과 똑같이 작동합니다.

터보 스크롤: 이 모드를 사용하면 클래식 스크롤보다 빠르게 스크롤할 수 있습니다. 또한 손가락을 떼지 않고도 여러 페이지를 스크롤할 수 있습니다. 스크롤 양은 스크롤을 시작한 후 손가락을 움직인 거리에 비례합니다.

떼면 스크롤: 이 스크롤 유형은 전자 잉크 화면에 더 적합합니다. 다른 스크롤 유형과 달리 이 모드는 스크롤하는 동안 페이지를 계속 새로고침하지 않기 때문입니다. 클래식 스크롤처럼 작동하지만 중간 단계가 표시되지 않습니다. 따라서 손가락으로 스크롤을 하다가 손을 떼면 페이지가 해당 위치로 이동합니다. 이 모드는 책에 그림이나 표가 포함되어 있어 모든 내용을 보려는 경우 보기 위치를 변경하고 조정하는 데 특히 유용합니다.

이전 위치로 돌아가기
이 유용한 기능은 브라우저의 뒤로 가기 버튼처럼 작동합니다. 훑어보기 위젯, 책갈피 목록, 책 지도 등을 사용하여 책의 다른 부분으로 이동할 때. 코리더는 이전 위치를 기억합니다. 이 기능은 예를 들어 내부 링크나 참조를 따라갈 때 유용합니다. 탭 한 번으로 어디에서 왔는지 돌아갈 수 있습니다. 이 기능은 다음에서 접근할 수 있습니다:
 상단 메뉴 → → 이전 위치로 돌아가기
또한 같은 메뉴에서 앞으로 이동하는 것도 가능합니다. 이 기능은 ← → 다중 스와이프 제스처에 할당되어 있습니다. 원하는 경우 다른 제스처(예: 모서리 탭)에 할당하거나 빠른 메뉴에 추가할 수 있습니다.
이미지가 있는 페이지에서 깜박임 방지하기
코리더는 일반적으로 이미지가 포함된 페이지에서 전체 새로 고침을 수행합니다(검은색 깜박임으로 확인할 수 있음). 그 이유는 이전 페이지의 잔여물과 고스트 현상을 제거하기 위해서입니다. 하지만 사용 중인 디바이스와 개인 취향에 따라 이 기능이 필요하지 않을 수도 있습니다. 이 자동 새로고침 동작은 여기에서 비활성화할 수 있습니다:
 상단 메뉴 → → 화면 → E-ink 설정 → 최대 새로 고침 빈도 → 이미지가 있는 페이지에서 항상 깜박임
부분 렌더링 지원
2023.03 버전에서 KOReader는 EPUB 문서를 읽는 동안 부분 렌더링 기능을 추가했습니다. 이 버전 이전에는 글꼴 크기를 변경하는 등 조정할 때 책 전체를 다시 렌더링해야 새 설정을 미리 볼 수 있었습니다. 장치와 책 크기에 따라 계산하는 데 시간이 걸릴 수 있습니다.

이 새로운 기능을 사용하면 EPUB 문서를 읽을 때 현재 챕터만 렌더링하여 텍스트 모양을 훨씬 더 빠르게 조정할 수 있습니다.

 이것은 EPUB이 여러 개의 조각으로 구성된 경우에만 가능합니다. EPUB 파일은 기본적으로 .html 파일이 포함된 .zip 아카이브입니다. 일부 책은 단일 .html 파일일 수도 있고, 책의 각 장이 별도의 .html 파일인 것처럼, 여러 .html 파일로 구성될 수도 있습니다. 책이 단일 .html 파일인 경우 변경할 때마다 KOReader가 전체 책을 계산해야 하므로, 부분 렌더링이 작동하지 않습니다.

또한 이 기능 덕분에 화면 방향을 회전하는 속도가 훨씬 빨라졌습니다. 이 기능은 기본적으로 활성화되어 있습니다. 필요한 경우 책별로(탭하여) 또는 전체적으로(길게 탭하여) 비활성화할 수 있습니다:

 상단 메뉴 → → 문서 → 부분 렌더링 활성화

이러한 부분 렌더링이 끝나면 코리더와 책은 성능이 저하된 상태가 됩니다: 페이지를 넘길 수는 있지만 일부 정보 및 기능(예: 바닥글 정보, 차례, 통계 등)이 깨지거나 비활성화될 수 있습니다. 정상적인 작동 상태로 되돌리려면 KOReader가 책의 나머지 부분도 렌더링해야 합니다. 이 작업이 진행되는 동안 화면 왼쪽 상단의 아이콘이 상태를 나타냅니다:

  1. 문서가 부분적으로 렌더링됩니다. 페이지 수와 바닥글 정보, 많은 부분이 정확하지 않습니다. 새 설정이 어떻게 보이는지 확인하고 추가로 조정할 수 있습니다. 이 상태에서 페이지를 넘기고 링크를 이동할 수도 있습니다.
  2. 전체 렌더링이 배경에서 이루어지고 있습니다. 여전히 페이지를 넘기고, 링크를 이동하고, 설정을 변경할 수 있습니다.
  3. 전체 렌더링이 완료되었지만 아직 적용되지 않은 이유는 KOReader가 책의 나머지 부분을 다시 로드하기 위해 사용자가 유휴 상태가 되를 기다리고 있기 때문입니다(유휴 상태=장치와 상호 작용하지 않음). 여전히 페이지를 넘기고, 링크를 이동하고, 설정을 변경할 수 있습니다.
  4. 잠시 동안 유휴 상태였으므로 이제 KOReader가 전체 문서를 다시 로드합니다. 이 단계에서는 상호 작용을 방지하기 위해 KOReader가 차단됩니다.
이 아이콘이 모두 사라지면 다시 정상 작동 상태가 됩니다. 일반적으로 이 모든 과정은 빠르게 진행되며 사용자는 추가 작업을 할 필요가 없습니다. 설정을 조정하고 책을 계속 읽기만 하면 나머지는 KOReader가 알아서 처리합니다.
하단 메뉴 항목 설명
 하단 메뉴에서 해당 항목의 이름을 길게 누르면 해당 항목에 대한 설명을 볼 수 있습니다.
보기 모드: 연속 모드로 변경하면 웹 페이지처럼 문서를 스크롤할 수 있습니다. 이 모드는 전자 잉크가 아닌 화면 유형(LCD, OLED)에 더 적합합니다. 연속 모드에서는 위쪽 및 아래쪽 여백이 사라지므로 화면의 첫 번째 줄과 마지막 줄에서 텍스트가 잘릴 수 있습니다. 이 문제를 해결하려면 페이지 겹침을 활성화하면 됩니다:
 상단 메뉴 → → 페이지 겹침
렌더링 모드: 이 옵션은 KOReader에서 문서를 렌더링하는 방식을 결정합니다. 기본값은 웹이며 일반적으로 이 설정을 변경할 필요가 없습니다. 책(또는 저장된 웹 페이지)이 매우 복잡하여(지정된 너비 또는 여백, 음수 여백, 플로트 등) 레이아웃이 좋지 않거나 화면이 넘쳐 보이는 경우 다른 렌더링 모드를 사용해 볼 수 있습니다.

확대/축소(dpi): 이 설정은 문서의 일반적인 확대/축소 비율로 생각할 수 있습니다(글꼴 크기 제외). 96dpi(기본 설정)에서는 문서의 이미지가 원래 크기로 렌더링됩니다. 기본적으로 텍스트를 더 크게 만들지 않고 이미지를 더 크게 만들려는 경우 이 설정을 사용합니다.

낱말 간격과 낱말 확장: 이 두 설정은 서로 연관되어 있으며 텍스트가 맞춤 설정되었을 때 텍스트가 어떻게 보이는지를 결정합니다. 텍스트를 맞춤 설정하면 낱말 사이에 공백이 추가되어 각 줄의 양쪽 가장자리가 양쪽 여백에 맞춰 정렬됩니다. 줄의 낱말 수가 항상 같지는 않으므로 맞춤 정렬 시 줄을 채우기 위해 텍스트를 압축하거나 확장해야 합니다.

낱말 간격: 더 많은 낱말을 넣기 위해 텍스트 한 줄의 공백 문자를 얼마나 압축할 수 있는지 설정합니다. 이 설정은 낱말에는 영향을 주지 않고 낱말 사이의 공백만 변경합니다.

낱말 확장: 한 줄에 너무 많은 공백이 있는 경우 공백을 확장하여 낱말에 분배할 수 있는 양을 설정합니다. 이 설정은 낱말의 모양에 영향을 줍니다. 낱말 확장을 원하지 않으면 이 설정을 없음으로 설정합니다.

텍스트가 마음에 들 때까지 다양한 설정을 실험해 본 다음 각 대화 상자를 통해 기본값으로 설정할 수 있습니다.

대비와 글꼴 굵기는 텍스트를 더 굵게(또는 더 가볍게) 만드는 데 사용할 수 있는 두 가지 옵션입니다. 이 옵션 중 하나를 사용하여 동일한 모양을 얻을 수 있지만 다른 방법을 사용합니다. 물론 두 가지를 동시에 사용할 수도 있습니다. 두 옵션의 차이점은 다음과 같이 요약할 수 있습니다:
대비

+ 글꼴 너비는 변경되지 않으며, 책 길이는 그대로 유지됩니다

+ 문서를 다시 렌더링할 필요가 없으며, 더 빠름

- LCD나 낮은 DPI 화면에서 보기 좋지 않을 수 있습니다

글꼴 굵기

+ 설치된 글꼴이 있는 경우 알맞은 굵기의 글꼴을 사용합니다

+ LCD나 낮은 DPI 화면에서 더 잘 보입니다

- 글꼴을 다시 렌더링해야 하며, 더 느림

글꼴 힌팅은 화면의 픽셀 그리드에 따라 가독성을 극대화하도록 텍스트를 조정합니다. 네이티브는 글꼴의 내부 힌트 지침을 사용하며, 자동은 FreeType의 힌트 알고리즘을 사용합니다. 설치한 글꼴 파일에 힌트 지침에 문제가 있을 수 있으므로 자동이 더 안전한 선택입니다. 또한 자동은 CJK 텍스트를 더 잘 처리합니다. 다른 설정을 시도하여 어떤 설정이 디바이스에서 더 잘 보이는지 확인할 수 있습니다.

글꼴 커닝은 시각적으로 보기 좋은 결과를 얻기 위해 글자 간 간격을 조정합니다. 기본값은 최상으로, 파일을 열 때 속도가 느려질 수 있지만(기기에 따라 다름) 합자(아래 그림에서 예시 참조), 아랍어 글리프와 일부 다른 스크립트를 더 잘 지원합니다. 기기가 느린 경우, 더 빠르지만 서양 라틴 문자 텍스트가 올바르게 표시될 수 있는 좋음을 사용해 볼 수 있습니다.

대체 상태 표시줄은 리플로우 가능한 문서(EPUB, HTML, DOCX, RTF, TXT...)에서만 사용할 수 있는 대체 상태 표시줄을 활성화합니다. 여기에서 활성화한 후 다음을 통해 대체 상태 표시줄을 구성할 수 있습니다:
 상단 메뉴 → → 상태 표시줄 → 대체 상태 표시줄
임베디드 스타일과 임베디드 글꼴은 문서 렌더링 시 문서에 임베디드된 발행인의 스타일과 글꼴을 사용할지(또는 무시할지) 결정합니다. 이는 일반적인 켜기/끄기 스위치입니다. 다음을 통해 문서를 보다 세밀하게 조정할 수 있습니다:
 상단 메뉴 → → 스타일 덧꾸밈
이미지 크기 조정은 문서의 이미지가 렌더링되는 방식을 결정합니다. 최상은 더 보기 좋게 보이지만 속도가 느립니다.
나중에 사용할 수 있도록 텍스트 모양 설정 저장하기
이 강력한 기능을 사용하면 모든 하단 메뉴 설정 + 글꼴 유형을 프로필로 저장할 수 있습니다. 즉, 매번 수동으로 설정을 변경하는 대신 다양한 조건이나 책 유형에 대해 서로 다른 설정 그룹을 만들어 한 번에 적용할 수 있습니다. 예를 들면:
  • 작은 크기의 텍스트, 가로 방향과 최소 여백으로 구성된 2단의 2단 프로필
  • 더 큰 산세리프 글꼴과 더 높은 대비로 어두운 곳에서 텍스트를 더 잘 볼 수 있는 야간 읽기 프로필
  • 잘못된 구조의 책에서 내장된 스타일과 내장된 글꼴이 비활성화된 스타일 프로필을 사용 않음
원하는 대로 설정을 조정한 후 다음을 통해 프로필을 만들 수 있습니다:
 상단 메뉴 → → 프로필 → 현재 문서 설정으로 새로 만들기
설정으로 프로필을 만든 후에는 이를 다른 책에 적용할 수 있는 다양한 옵션이 있습니다:
  • 프로필 메뉴에서 프로필을 적용할 수 있습니다
  • 제스처를 할당할 수 있습니다
  • 빠른 메뉴에 표시할 수 있습니다

자세한 내용은 제스처와 빠른 메뉴, 프로필 장을 참조하세요.

     하단 메뉴 →
  • 글꼴을 더 굵게 만들기

     상단 메뉴 → → 스타일 덧꾸밈
 *글꼴 선택과 스타일 조정은 가변형 문서(EPUB, HTML, DOCX, RTF, TXT...)에서만 사용할 수 있습니다. PDF/DJVU와 같은 고정형 문서에서는 지원되지 않습니다.

하단 메뉴에서 다른 서식 옵션을 변경하여 텍스트의 모양을 미세 조정할 수도 있습니다.

 상단 메뉴 → → 읽기 통계
화면의 배경 조명/전면 조명을 제어하는 방법?
이 메뉴를 통해 화면 조명을 제어할 수 있습니다. 따뜻한 조명(일반 흰색 LED+주황색 LED)이 있는 경우 이 대화 상자에서 별도로 제어할 수 있습니다. 이 대화 상자에 대한 제스처를 정의할 수도 있습니다:
 상단 메뉴 → → 전면 조명
상태 표시줄은 어떻게 구성하나요?
이 메뉴에서 하단 상태 표시줄을 구성할 수 있습니다. 설정 하위 메뉴를 자유롭게 탐색하여 원하는 대로 상태 표시줄을 사용자화 할 수 있습니다.
 상단 메뉴 → → 상태 표시줄
상태 표시줄에 표시할 수 있는 항목은 여러 가지가 있습니다. 하지만 기본적으로 한 번에 하나씩만 표시되며 상태 표시줄을 클릭해야 항목을 전환할 수 있습니다. 모든 항목을 한 번에 표시하려면 이 메뉴 항목을 활성화하세요:
 상단 메뉴 → → 상태 표시줄 → 설정 → 한 번에 모두 보기
다음과 같이 항목을 정렬할 수 있습니다:
 상단 메뉴 → → 상태 표시줄 → 설정 → 항목 정렬
제스처를 사용하거나 빠른 메뉴를 통해 상태 표시줄의 표시 여부를 켜고 끌 수 있습니다:
제스처 선택 메뉴 → 리더 → 상태 표시줄 토글
가변형 문서(epub, html, docx, rtf, txt...)에서 사용할 수 있는대체 상태 표시줄도 있습니다. 이 기능을 사용하려면 다음으로 활성화할 수 있습니다:
 하단 메뉴 → → 대체 상태 표시줄
잠자기 상태에서 표시되는 화면을 변경하는 방법?
이 메뉴를 통해 책 표지나 사용자 지정 이미지를 화면 보호기로 선택할 수 있습니다:
 상단 메뉴 → → 화면 → 화면보호기

JPG/PNG 파일을 화면 보호기로 사용할 수 있습니다(적절하게 디더링된 회색조 이미지가 가장 잘 보입니다). 컴퓨터에서 디바이스의 임의 폴더로 파일을 전송합니다. 그리고 이 메뉴에서 이 폴더를 화면 보호기 폴더로 선택합니다:

 상단 메뉴 → → 화면 → 화면보호기 → 설정
화면 보호기 이미지에 대한 고급 정보는 이 포럼 게시물을 참조하세요: https://www.mobileread.com/forums/showpost.php?p=3728291&postcount;=17
키보드 사용법과 팁

  • 키를 위로 스와이프하여 해당 문자의 대문자 버전(이미 시프트 모드인 경우 소문자)을 입력합니다

  • 키보드의 아이콘을 탭하여 선택한 배열 중에서 변경할 수 있습니다
특수 문자 입력
코리더 키보드를 사용하면, 키를 길게 눌러 특수 문자를 입력할 수 있습니다. 가장 자주 사용되는 특수 문자는 다음과 같습니다. 회색 상자 안의 키를 길게 누르면 주변 문자를 가져올 수 있습니다: 또한 더 쉬운 텍스트 조작을 위한 몇 가지 단축키가 있습니다. 다음 키를 길게 탭하여 작업할 수 있습니다:
  • 백스페이스 - 커서부터 줄 시작 부분까지의 모든 문자를 삭제합니다
  • 왼쪽 화살표 - 커서를 줄의 시작 부분으로 이동합니다
  • 오른쪽 화살표 - 커서를 줄 끝으로 이동합니다

자동 색 온도와 야간 모드
이 플러그인을 사용하면, 자연광(노랑/주황) 조명을 사용하는 장치의 전면 조명 색 온도를 조정하고 일정에 따라 자동으로 야간 모드(화면 색깔 반전 - 검은색 배경에 흰색 글씨)를 설정할 수 있습니다. 두 가지를 모두 제어하거나, 색 온도 제어 모드만이거나 야간 모드 제어 모드만 선택할 수 있으므로, 다른 하나는 수동으로 조정할 수 있습니다.

이 플러그인은 기본적으로 활성화되어 있습니다. 메뉴에 없는 경우 다음을 통해 활성화할 수 있습니다:

 상단 메뉴 → → 추가 도구 → 자동 색온도와 야간 모드
두 가지 방법으로 화면 전면 조명의 색 온도를 자동으로 조절할 수 있습니다:

고정된 일정: 이 모드에서는 입력한 일일 일정에 따라 전면 조명의 색 온도가 변경됩니다. 이것은 정적인 모드이며 낮의 길이를 고려하지 않습니다. 이 일정은 일년 내내 동일하게 유지됩니다.

태양의 위치: 이 옵션은 내가 입력한 좌표와 고도 정보에 따라 내 위치의 태양 위치를 계산합니다. 이 계산된 태양 위치 데이터에 따라 전면 조명의 색 온도가 점진적으로 조정됩니다. 이것은 동적인 모드입니다. 일년 내내 낮의 길이 변화에 따라 플러그인이 일정을 조정합니다.

먼저 이 이미지를 살펴보고 플러그인에서 사용되는 용어를 숙지하세요:

고정된 일정 모드
  1. 플러그인 메뉴로 이동합니다:
     상단 메뉴 → → 화면 → 자동 색 온도와 야간 모드
  2. 메뉴 항목 활성화를 탭한 다음 고정된 일정대로를 탭하세요
  3. 이전 메뉴로 돌아가서 고정된 일정 설정 메뉴 항목을 탭하세요
  4. 여기에서 다양한 항목의 시간을 설정하여 색 온도 변경 일정을 설정하세요. 항목 설명은 위 이미지를 참조하세요.
태양의 위치 모드
  1. 플러그인 메뉴로 이동합니다:
     상단 메뉴 → → 화면 → 자동 색 온도와 야간 모드
  2. 활성화 메뉴 항목을 탭한 다음 태양의 위치에 따라를 누르세요
  3. 이전 메뉴로 돌아가서 위치 설정 메뉴 항목을 탭합니다. 좌표(필수)와 고도(선택)를 올바르게 설정하고 내 위치의 이름을 지정합니다.
  4. 이전 메뉴로 돌아가서 색 온도와 야간 모드 설정 메뉴 항목을 탭하세요
  5. 여기에서 다양한 항목의 시간을 설정하여 색 온도 변경 일정을 설정하세요. 항목 설명은 위 이미지를 참조하세요.

현재 활성화된 매개변수 메뉴 옵션을 통해 현재 일정을 확인할 수 있습니다.

야간 모드
다양한 간격으로 색 온도 조명 비율을 설정하는 동안, 해당 간격을 야간 모드로 설정하여 해당 간격이 시작되면 화면 색상이 반전되도록 할 수도 있습니다. 최소 전면 조명이 여전히 너무 밝거나 어두운 페이지 배경을 선호하는 경우, 이 기능을 활성화할 수 있습니다. 이 기능을 활성화하려면 오른쪽 대화 상자에서 야간 모드 사용을 탭하면 됩니다.

이렇게 하면 색 온도 조명 비율이 110%로 증가하여 이 간격이 야간 모드가 활성화되었음을 나타냅니다. 이 간격 옆의 메뉴에 달 아이콘이 표시됩니다. 야간 모드를 비활성화하려면 이 비율을 110 미만의 값으로 낮추면 됩니다.

야간 모드를 수동으로 관리하려면(또는 그 반대) 아래 표시된 옵션을 탭하여 이 플러그인의 작동 모드를 변경할 수 있습니다.

전면 조명 자동 켜기/끄기
이 기능은 이 플러그인의 현재 활성화된 매개변수에 따라 일몰 때 전면 조명을 켜고 일출 때 전면 조명을 끕니다. 수동으로 전면 조명을 켜거나 꺼서 이 변경 사항을 재정의할 수 있습니다. 다음 일출/일몰 때 자동 색온도가 필요한 경우 다시 토글됩니다. 이 플러그인에서 낮 동안 전면 조명 끄기 메뉴 항목을 선택하여 활성화할 수 있습니다.

KOReader는 가장 널리 사용되는 트루타입(.ttf)과 오픈타입(.otf) 형식의 글꼴을 지원합니다. 이러한 확장자를 가진 글꼴 파일은 내 기기의 /koreader/fonts/ 폴더에 직접 복사할 수 있습니다. 자세한 내용은 위키를 참조하세요:

https://github.com/koreader/koreader/wiki/Fonts

KOReader는 시작할 때 글꼴 메타데이터/정보에서 사용 가능한 글꼴을 컴파일하고 패밀리와 굵기에 따라 글꼴을 분류합니다. 즉, 글꼴의 다양한 굵기와 변형을 추가할 수 있으며, KOReader는 가장 적합한 글꼴을 사용하려고 시도합니다. 메타데이터에서 글꼴 패밀리 정보를 가져오기 때문에 내 기기에 복사할 때 글꼴 이름은 중요하지 않습니다. 설치된 글꼴은 KOReader를 다시 시작한 후에 사용할 수 있습니다.

Google 글꼴 웹사이트에서 무료 글꼴을 받을 수 있습니다: https://fonts.google.com

내 기기에서 글꼴 미리보기

KOReader는 기기에 설치된 모든 글꼴로 렌더링된 샘플 텍스트가 포함된 문서를 생성할 수 있습니다. 이를 통해 그리스어, 히브리어 또는 중국어와 같은 비 라틴어 스크립트의 합자 및 문자 세트와 같은 글꼴의 모양과 기능을 빠르게 비교할 수 있습니다. 이 기능은 다음을 통해 접근할 수 있습니다:

 상단 메뉴 → → 글꼴 → 글꼴 설정 → 글꼴 테스트 문서 생성
고정폭 글꼴 크기

프로그래밍 책에서처럼 책에 고정폭 글꼴이 있는 부분이 포함되어 있는 경우, 기술적으로 크기가 같더라도 고정폭 글꼴이 불균형하게 보일 수 있습니다. 이는 글꼴 메트릭이 다르기 때문에 발생합니다. 이 차이를 해결하려면 KOReader에 상대적인 고정폭 글꼴 크기를 조정할 수 있는 옵션이 있습니다.

 상단 메뉴 → → 글꼴 → 글꼴 설정 → 고정폭 글꼴 크기 조정

여기에서 기본 크기의 백분율로 배율을 조정하여 고정폭 글꼴의 크기를 늘리거나 줄일 수 있습니다.

한 문서에서 다른 글꼴 사용

EPUB과 같은 HTML/CSS 기반 문서에서, 책은 특정 글꼴 이름 대신 글꼴 패밀리 별로 글꼴을 사용하도록 지정할 수 있습니다. 이러한 글꼴 패밀리는 세리프, 산세리프, 고정폭, 필기체, 판타지, 이모티콘, 팡송, 수학식 등이 있습니다. 예를 들어 책에서 일반 텍스트는 산세리프로 표시하고, 대화 상자는 세리프로 표시할 수 있습니다. KOReader는 일반적으로 고정폭과 수학식을 제외한 모든 글꼴 패밀리(텍스트 노드의 스타일에 따라 결정되는 'font-family: serif' 등)에 기본 글꼴을 사용합니다. 이러한 글꼴 패밀리가 뚜렷하게 보이도록 하려면, 여기에서 특정 글꼴을 지정할 수 있습니다:

 상단 메뉴 → → 글꼴 → 글꼴 패밀리 글꼴

현재 책의 패밀리 글꼴을 변경하려면 원하는 글꼴 이름을 탭하면 됩니다. 모든 책에 설정하려면 길게 탭하여 기본값(★)으로 설정합니다. 책에서 기본 글꼴이 거의 사용되지 않을 정도로 이 글꼴 패밀리 기능을 남용하는 경우, 연결을 해제하여 이 책에 대해 해당 패밀리 글꼴 기능을 비활성화할 수 있습니다.

훑어보기 위젯

챕터 표시

문서에 제목 단계가 너무 많으면, 왼쪽의 예처럼 챕터 표시가 복잡해 보일 수 있습니다. 다음을 통해 표시되는 차례 표시 단계의 깊이를 변경할 수 있습니다:

 상단 메뉴 → → 설정 → 진행 막대

이 감소된 차례는 다음 용도로도 사용할 수 있습니다:

이러한 옵션은 동일한 메뉴에 있습니다.

 상단 메뉴 → → 설정 → 대체 차례
책 지도
책 지도는 책의 개요를 파악하는 데 매우 유용한 KOReader의 강력하고 독특한 기능입니다. 예를 들어, 책을 읽다가 일부 부분을 건너뛰고 나중에 읽는 등 비선형적으로 읽는 경우 진행 상황을 추적하기 어려울 수 있습니다. 책 지도를 사용하면 책의 어느 부분을 이미 읽었는지 한 눈에 확인할 수 있습니다.

추가적으로 가능합니다:

  • 각 페이지에서 시간을 얼마나 보냈는지 확인하세요(읽기 통계 플러그인이 활성화된 경우)
  • 책갈피, 강조 표시, 메모가 있는 페이지 확인
  • 훑어보기 위젯과 같이 책의 원하는 부분으로 바로 탐색
메뉴를 통해 책 지도에 접근할 수 있으며, 더 빠르게 접근할 수 있도록 제스처를 지정할 수도 있습니다:
 상단 메뉴 → → 책 지도
사용자 인터페이스와 제어
책을 읽다가 책 지도를 처음 열면, 격자 보기 모드가 되어 모든 챕터를 한 화면에 표시하므로, 책 내용에 대한 개요를 가장 잘 파악할 수 있습니다. 사용법에 대한 자세한 정보를 보려면, 왼쪽 상단의 정보 아이콘을 탭하세요. 오른쪽 상단 모서리에 있는 X를 클릭하거나 멀티스와이프 제스처(위-아래, 왼쪽-오른쪽 등)로 책 지도를 닫을 수 있습니다. 이러한 옵션을 변경하여 책 지도를 사용자화 할 수 있습니다:

- 먼저 화면 하단 가장자리를 따라 왼쪽과 오른쪽을 스와이프하여 페이지를 나타내는 열의 너비를 변경하고 지도의 축척이 어떻게 변하는지 관찰하세요

- 그런 다음 화면 왼쪽 모서리를 따라 위쪽과 아래쪽으로 스와이프하여 챕터 제목의 깊이를 변경하고 평면 보기로 전환합니다. 아래 예를 참조하세요.

위의 기본 격자 보기 외에도, 왼쪽에는 평면 보기가, 오른쪽에는 챕터 제목이 없는 격자 보기가 표시됩니다. 보기 간에 왼쪽 세로 스크롤 막대가 변경되는 것을 확인할 수 있습니다. 기본 보기로 다시 전환하려면 왼쪽 위 모서리에 있는 정보 아이콘을 길게 누르면 됩니다.

책 지도 선택 사항
책 지도 화면에서 메뉴 아이콘을 탭하면 위와 같은 메뉴가 표시됩니다. 메뉴 옵션은 아래에 설명되어 있습니다:
  • 책 지도 정보: 책 지도 사용법과 아이콘에 대한 간략한 설명입니다.
  • 가능한 제스처: 이 메뉴를 사용하는 대신 책 지도의 설정을 변경하는 데 사용할 수 있는 제스처입니다.
  • 탭하면 페이지 탐색기: 이 옵션을 선택하면 책 지도에서 페이지를 탭하면 페이지 탐색기가 열립니다. 여기에는 선택한 영역의 개요가 표시됩니다. 이 옵션을 비활성화하면, 책 지도를 탭하면 해당 페이지로 바로 이동합니다. 하지만 화면 크기, 터치 스크린 감도, 탭 정확도 등과 같은 요인에 따라 인접 페이지로 이동할 수도 있습니다.
  • 현재/시작 보기 전환: 책 지도 보기를 사용자 지정한 경우, 이 기능을 탭하면 기본 보기와 사용자 지정 보기 간에 전환됩니다.
  • 격자/평면 보기 전환: 개별 챕터나 연속 챕터 보기 간에 전환합니다.
  • 챕터 단계: 챕터/하위 챕터 단계의 깊이를 변경합니다.
  • 페이지 슬롯 너비: 각 페이지 슬롯을 더 넓게 또는 더 좁게 설정합니다.
  • 10페이지 표시: 이 값을 늘리면 처음에는 10번째 페이지마다 표시를 추가하고, 그 다음에는 더 크게 만들고, 결국에는 5번째 페이지마다 표시를 추가하게 됩니다.
페이지 탐색기
페이지 탐색기는 페이지의 썸네일을 격자로 표시하여 사진 앨범처럼 페이지 사이를 탐색하는 데 사용할 수 있습니다. 책 지도를 탭하거나 메뉴에서 페이지 탐색기를 열 수 있습니다. 빠른 접근을 위해 제스처를 지정할 수도 있습니다.
 상단 메뉴 → → 페이지 탐색기
화면 위쪽과 왼쪽을 따라 스와이프하여 표시되는 페이지 수를 변경할 수 있습니다. 하단의 아이콘은 책 지도와 동일합니다. 사실 하단은 책 지도의 직선형 버전입니다.
책갈피와 강조 표시, 메모
KOReader는 실제 책에서 하는 것처럼 책과 상호작용할 수 있는 선택 사항을 제공합니다:

책갈피: 오른쪽 상단 모서리를 눌러 책갈피를 추가(및 제거)할 수 있습니다. 이는 실제 책에서 페이지의 모서리를 접는 것과 같으며 활성화하면 비슷한 아이콘이 표시됩니다.

강조 표시: 4가지 스타일로 글씨에 강조 표시를 할 수 있습니다. 아래 이미지에서는 일반 문단 뒤에 다양한 스타일의 강조 표시가 있는 것을 볼 수 있습니다. 밝게(회색 배경)와 밑줄, 취소선 , 반전.

이 모든 스타일을 서로 다른 색깔의 강조 표시 마커처럼 동시에 사용할 수 있습니다. 또는 이 메뉴를 길게 눌러 좋아하는 스타일을 결정하고 기본값으로 설정할 수도 있습니다:

 상단 메뉴 → → 강조 표시 스타일
메모: 강조 표시에 메모를 추가할 수도 있습니다. 강조 표시를 탭하고 팝업 메뉴에서 메모 추가를 선택하면 됩니다. 메모가 포함된 강조 표시를 보여주는 메모 표시를 선택할 수도 있습니다. 세 가지 스타일이 있습니다: 밑줄, 옆줄, 측면 표시(아래 그림의 오른쪽에 보이는 연필 아이콘): 이 기능을 활성화하고 메모 표시 스타일을 선택하려면 다음으로 이동하세요:
 상단 메뉴 → → 강조 표시 스타일 → 메모 표시자
강조 표시하는 방법
KOReader에는 다양한 사용 사례를 위한 두 가지 강조 표시 모드가 있습니다:

일반 모드는 휴대폰/태블릿에서 손가락으로 내용을 선택하는 것과 유사하며, 강조 표시가 한 페이지보다 짧은 경우에 더 적합합니다.

내용 선택과 강조 표시 모드는 페이지 경계를 넘나드는 긴 구절을 강조 표시하는 데 더 적합합니다. 터치 스크린 감도 문제나 손의 운동 제어 문제가 있는 경우에도 이 모드가 더 효과적일 수 있습니다. 이 강조 표시 모드에서는 화면을 따라 손가락을 드래그할 필요가 없기 때문입니다.

일반 모드
이 모드는 휴대폰/태블릿에서 손가락으로 문자를 선택하는 것과 비슷합니다.
  1. 시작 단어를 길게 누르세요
  2. 손가락을 구절 끝으로 이동하세요
  3. 손가락 떼기
페이지 경계에 다다르면 KOReader가 페이지를 넘어서(EPUB/HTML 문서에만 해당) 계속 강조 표시를 합니다.
내용 선택과 강조 표시 모드
 단어를 길게 누를 때 사전 검색을 활성화한 경우, 이 모드를 시작하려면 손가락을 드래그해야 합니다.
 상단 메뉴 → → 탭과 제스처 → 텍스트 길게 누르기 → 한 낱말 선택하여 사전 열기
  1. 시작 단어를 길게 누르면(또는 위에서 설명한 대로 사전 조회가 활성화된 경우 드래그하면) 팝업 메뉴가 표시됩니다.
  2. 선택을 탭하면 강조 표시 모드가 됩니다. 이 모드에서는 화면 왼쪽 상단에 표시자가 표시되며, 이 표시자를 탭하면 자세한 정보를 볼 수 있습니다. 강조 표시 시작 부분을 다시 탭하여 강조 표시를 취소할 수 있습니다. 이 모드에서는 탭/스와이프하여 페이지를 넘기고 문서를 자유롭게 탐색(차례, 훑어보기 등)할 수 있습니다.
  3. 강조 표시의 끝 단어를 길게 누르면(또는 위에서 설명한 대로 사전 조회가 활성화된 경우 드래그하면) 메뉴가 다시 나타납니다.
  4. 이번에는 강조 표시를 탭하여 강조 표시 구절을 끝냅니다.
 위에 표시된 메뉴 항목을 길게 누를 필요가 없는 경우, 이 모드를 더 빠르게 사용할 수 있는 방법이 있습니다. 팝업 메뉴를 완전히 비활성화하면, 낱말을 길게 누르면 즉시 선택 후 강조 표시 모드로 진입합니다. 이 방법은 훨씬 빠르며 두 번만 탭하면 어떤 길이의 텍스트도 강조 표시할 수 있습니다. 이 방법을 사용하려면 이 옵션을 활성화합니다:

강조 표시할 때 KOReader는 선택 영역 주위에 관련 문장 부호를 포함하려고 시도합니다. 예를 들어 문장의 첫 낱말부터 마지막 낱말까지 강조 표시하면, KOReader는 문장 양쪽 끝에 있는 문장 부호를 자동으로 강조 표시 영역에 포함합니다.

그러나 문장 중간부터 강조 표시를 시작하면, KOReader는 사용자가 부분적으로 강조 표시를 하려는 것으로 간주하므로 끝의 마침표가 포함되지 않습니다. 따라서 문장 부호를 강조 표시할 때, 문장 부호를 추가하려면 문장 전체를 선택해야 합니다. 또한 나중에 강조 표시 편집 버튼을 길게 눌러 이 부호들을 추가할 수도 있습니다.

빠른 메뉴와 프로필
빠른 메뉴
빠른 메뉴는 빠르게 접근할 수 있는 개인화된 메뉴를 제공하는 KOReader의 최신 기능입니다. 이 메뉴를 제스처에 할당하고 한 번 스와이프하여 접근할 수 있습니다. 오른쪽에서 메뉴 예시를 볼 수 있습니다. 화면보다 긴 메뉴를 만들 수도 있고, 스크롤이 가능합니다.

기본적으로 제스처에 할당할 수 있는 모든 동작을 빠른 메뉴에 추가할 수 있습니다. 장점은 다음과 같습니다:

  1. 모든 동작을 위해 많은 제스처를 외울 필요는 없습니다.
  2. 일부 자주 사용하지 않는 동작은 제스처의 가치가 없을 수도 있지만 메뉴 시스템에서 찾기에는 너무 번거로울 수 있습니다. 이런 종류의 자주 사용하지 않는 동작을 대신하여 이 메뉴에 추가할 수 있습니다.
  3. 친척이나 친구를 위해 KOReader를 설치하는 경우, 이 메뉴에 그들이 좋아하는 작업을 추가하면 KOReader를 사용하는 동안 메뉴 시스템으로 이동할 필요도 없습니다.

빠른 메뉴 예시:

빠른 메뉴를 만드는 방법에는 두 가지가 있습니다. 쉬운과 고급. 메뉴가 하나만 필요한 경우 쉬운 방법으로 충분합니다. 다른 제스처로 다른 메뉴를 표시하려면 고급 방법을 사용해야 합니다. 고급 방법의 또 다른 장점은 메뉴에 다른 제스처를 사용하려는 경우 다른 제스처로 쉽게 이동할 수 있다는 것입니다. 쉬운 방법을 사용하는 동안 다른 제스처를 사용하려면 새 제스처로 메뉴를 다시 만들어야 합니다.

쉬운 방법
  1. 다음을 통해 제스처 관리자 메뉴로 이동합니다:
     상단 메뉴 → → 탭과 제스처 → 제스처 관리자
  2. 메뉴를 여는 제스처를 결정하고 탭합니다.
  3. 제스처 구성 메뉴에서 메뉴에 표시할 동작을 선택합니다. 물론 한 번에 여러 동작을 선택할 수도 있습니다.
  4. 모든 동작을 선택한 후, 이 제스처를 메뉴로 취급하도록 KOReader에 지시합니다. 아래 그림에서 볼 수 있듯이 두 번째 페이지에서 빠른 메뉴로 보기 항목을 활성화하면 됩니다.
  5. 메뉴 항목 바로 아래에 있는 정렬 옵션을 사용하여 메뉴 항목 순서를 정렬할 수 있습니다.

이제 선택한 제스처로 메뉴를 실행할 수 있습니다. 메뉴 항목을 변경하려면 제스처 페이지를 열고 항목을 추가/제거하면 됩니다.

고급 방법
 위에서 설명했듯이 빠른 메뉴를 두 개 이상 추가할 계획이 없는 경우에는 이 섹션이 필요하지 않습니다.

빠른 메뉴는 실제로 화면에 메뉴로 표시되는 프로필입니다. 따라서 먼저 프로필에 대해 배워야 합니다. 프로필 사용법을 이미 알고 있다면 계속 읽어도 되지만, 그렇지 않다면 프로필 표제(다음 섹션)로 이동하여 몇 가지 항목으로 프로필을 만든 다음 여기로 돌아와도 됩니다.

이전 문단의 안내에 따라 이미 프로필을 만들었다고 가정합니다. 이제 프로필을 빠른 메뉴로 전환하겠습니다.

1. 새로 만든 프로필로 이동하여 동작 목록에 표시를 활성화합니다(아래 이미지 참조). 이렇게 하면 제스처 관리자에서 프로필이 표시됩니다. 또한 프로필이 자동으로 실행되는 대신 메뉴로 표시되도록 빠른 메뉴로 표시(아래 이미지에도 표시됨)도 활성화합니다.

2. 이제 메뉴에 액세스할 수 있는 방법이 필요합니다. 제스처 관리자로 이동하여 목록에서 메뉴에 할당할 제스처를 선택합니다. 화면 모서리를 탭하여 메뉴를 표시하고 싶다고 가정해 보겠습니다. 이제 모서리 탭을 탭한 다음 왼쪽 하단을 선택합니다. 이제 일반을 탭하고 두 번째 페이지로 이동합니다. 프로필이 프로필 x로 표시되며, x는 사용자가 이 프로필에 지정한 이름입니다. 해당 확인란을 탭하여 이 제스처에 프로필을 할당합니다.

모든 단계를 마치면 이제 왼쪽 하단을 탭하면 메뉴가 표시됩니다. 항목을 추가하거나 제거할 수 있는 프로필 페이지로 이동하여 메뉴를 편집할 수 있습니다. 또한 정렬 항목이 있어 메뉴 항목의 순서를 변경할 수 있습니다.

앞서 말했듯이 빠른 메뉴는 사실 프로필입니다. 즉 두 개 이상의 메뉴를 가질 수 있습니다. 원하는 경우 다른 프로필을 만들고 동일한 단계에 따라 다른 메뉴를 만들 수 있습니다.

프로필
프로필은 기본적으로 한 번의 탭으로(매크로처럼) 트리거(실행)할 수 있는 작업의 모음입니다.

예를 들어, 어떤 유형의 책을 읽을 때 항상 다음과 같은 상황을 상상해 보세요:

  • 화면을 가로 방향으로 회전하기
  • 글꼴 크기와 대비를 늘리고
  • 화면 조명을 켜기

프로필 플러그인을 사용하면, 한 번 작업으로 이 모든 일을 한 번에 수행할 수 있습니다.

 다른 프로필을 활성화하면 첫 번째 프로필에서 변경한 내용은 되돌릴 수 없습니다. 예를 들어 글꼴 크기를 20에서 24로 설정하는 프로필을 활성화합니다. 그런 다음 전면 조명을 조정하는 다른 프로필을 활성화합니다. 두 번째 프로필을 활성화한 후에도 글꼴 크기는 24로 유지됩니다. KOReader의 프로필에는 끝내기나 되돌리기 동작이 없기 때문입니다.

프로필을 만들려면 먼저 프로필 플러그인이 활성화되어 있는지 확인하세요. 아래에 표시되지 않는 경우:

 상단 메뉴 → → 프로필

그러면 다음에서 켜야 합니다:

 상단 메뉴 → → 추가 도구 → 플러그인 관리
프로필 만들기
프로필 메뉴에서 새로 만들기를 탭하고 프로필에 설명이 포함된 이름을 입력합니다. 이제 메뉴에 새 프로필이 표시됩니다. 프로필을 탭하면 왼쪽에 화면이 표시됩니다. 이 메뉴에서 프로필을 삭제, 이름 바꾸기 또는 복제할 수도 있습니다.

동작 수정 메뉴 항목을 탭하면 오른쪽에 메뉴가 표시됩니다. 이 메뉴는 제스처 선택 메뉴와 동일한 메뉴이며 다양한 범주의 동작이 있습니다. 즉 프로필을 사용하여 모든 제스처 동작을 수행할 수 있습니다.

이제 이 모든 범주에서 여러 항목을 선택할 수 있습니다. 여기서 선택한 작업은 이 프로필을 활성화할 때 수행됩니다.

프로필 활성화하기
프로필을 활성화하는 여러 가지 방법:
  1. 동일한 프로필 메뉴에서 이름을 길게 눌러 프로필을 활성화할 수 있습니다
  2. 제스처를 사용하여 프로필을 활성화할 수 있습니다. 이를 위해서는 프로필 메뉴에서 작업 목록에서 보기를 활성화해야 합니다. 이제 다음을 통해 선택할 수 있습니다:
     상단 메뉴 → → 탭과 제스처 → 제스처 관리자 → "어떤 제스처" → 일반 → "내 프로필 이름" 프로필
  3. 프로필 메뉴에서 자동 시작을 활성화하면 KOReader가 시작될 때 프로필을 자동으로 활성화할 수도 있습니다. 이 기능은 KOReader가 파일 탐색기나 최근 파일로 시작될 때만 가능합니다.
프로필과 빠른 메뉴의 기술적 세부 사항 고급
KOReader에는 작업/설정 묶음이 있으며, 이 묶음은 두 가지 방법으로 만들 수 있습니다:
  1. (고급 방법으로 설명되고 settings/gestures.lua에 저장된) 프로필로
  2. (쉬운 방법으로 설명되고 settings/gestures.lua에 저장된) 제스처에 할당된 동작 목록으로

이 묶음은 두 가지 방법으로 실행할 수도 있습니다:

  1. (프로필로) 트리거(실행)될 때 (정렬된) 순서대로 각 작업을 적용
  2. 빠른 메뉴로 보여주기

묶음 동작을 추가/제거하고 빠른 메뉴를 활성화/정렬하는 작업은 동일한 동작 메뉴를 통해 수행됩니다. 유일한 차이점은 제스처 관리자의 동작 메뉴에 두 가지 항목(통과하는)이 더 있다는 점이며, 이 때문에 빠른 메뉴로 표시와 정렬 항목이 두 번째 페이지에 있습니다.

플러그인과 사용자 패치
플러그인 종류
플러그인은 KOReader의 기능을 확장하는 작은 프로그램입니다. 브라우저 애드온과 비슷하다고 생각하시면 됩니다. KOReader에는 3가지 범주의 플러그인이 있습니다:
내부의
내부 플러그인은 핵심 프로그램의 일부입니다.
  • KOReader 플러그인 메뉴에서 바로 접근할 수 있습니다.
  • 설치가 필요하지 않습니다. 메뉴에서 활성화할 수 있습니다.
  • 개발자가 직접 확인하므로 안전한 것으로 간주됩니다.
  • KOReader를 업데이트할 때 자동으로 업데이트됩니다.

다음에서 활성화/비활성화할 수 있습니다:

 상단 메뉴 → → 추가 도구 → 플러그인 관리
기여  고급
이 그룹에는 KOReader의 핵심 로직에 속하지 않는 비공식 플러그인이 포함되어 있습니다.
  • 수동으로 설치한 다음, 플러그인 메뉴에서 활성화해야 합니다.
  • 수동으로 업데이트해야 합니다.
  • 이러한 파일은 저장소에 추가되는 시점에 개발자가 확인하므로 안전한 것으로 간주됩니다.
이 페이지에서 액세스할 수 있습니다: https://github.com/koreader/contrib 위 페이지에서 플러그인 디렉토리를 다운로드한 후 기기의 koreader/plugins 폴더에 복사하여 설치할 수 있습니다. 하지만 다른 지침은 플러그인 페이지에서 확인하세요.
외부의  고급
KOReader 팀에서 독립적으로 개발한 플러그인입니다.
  • 수동으로 설치한 다음, 플러그인 메뉴에서 활성화해야 합니다.
  • 수동으로 업데이트해야 합니다.
  • 개발자가 확인하지 않았으므로 사용할 때 위험이 따를 수 있다는 점에 유의하세요.

이러한 외부 플러그인의 몇 가지 주목할 만한 예입니다:

  • AskGPT: 책의 일부분에 대해 ChatGPT에 질문할 수 있습니다. 이 플러그인을 사용하여 텍스트를 단순화, 설명, 토론하거나 다른 언어로 번역할 수도 있습니다. https://github.com/drewbaumann/AskGPT
  • KOReader용 Anki 플러그인: 내부 사전에서 검색된 낱말에 대해 Anki(오픈 소스 플래시카드 소프트웨어) 카드를 생성할 수 있는 KOReader 플러그인. https://github.com/Ajatt-Tools/anki.koplugin
  • KOReader용 Zotero: 이 애드온을 사용하면 KOReader에서 Zotero(오픈 소스 참조 관리 소프트웨어) 컬렉션을 탐색할 수 있습니다. https://github.com/stelzch/zotero.koplugin

또한 이 링크를 클릭하면 Github의 검색 페이지가 열리고 더 많은 KOReader 플러그인 프로젝트를 찾을 수 있습니다: https://github.com/search?q=koplugin&type;=repositories

자동 일시 중단 - 선택한 기간 동안 사용하지 않으면 기기를 일시 중단합니다(안드로이드 OS를 제외한 모든 전자 잉크 장치에서 사용 가능)

플래시카드 트레이너 - 보관함에서 강조 표시의 간격을 두고 반복하여(SM2) 플래시카드를 만드는 방법

제스처 - KOReader용 제스처 지원 제공

사용자 패치  고급
사용자 패치는 Lua 언어로 작성된 작은 파일입니다. 사용자 패치는 설치를 영구적으로 수정하지 않고도 KOReader의 기능을 변경할 수 있습니다. 개발자나 다른 사용자가 작성할 수도 있고, 사용자가 직접 작성할 수도 있습니다(KOReader 내부에 대한 광범위한 지식이 있는 경우).

사용자 패치의 최신 목록은 관련 GitHub 위키 페이지에서 확인할 수 있습니다:

https://github.com/koreader/koreader/wiki/User-patches

그리고 사용자 패치 사용 가능 태그 아래의 이슈 목록에 표시됩니다:

https://github.com/koreader/koreader/issues?q=+label%3A%22User+patch+available%22

기본 사용법 - 이미 작성된 패치를 KOReader에 추가합니다:

  1. 내 기기의 koreader 디렉터리 아래에 patches 라는 이름의 디렉터리를 만듭니다(안드로이드는 이 폴더가 SD Card 안에 있습니다)
  2. 이 patches 디렉터리에 확장자가 .lua인 패치 파일을 다운로드하여 넣습니다
  3. KOReader 재시작

패치 관리자

지침에 따라 패치 디렉토리를 생성하면 KOReader를 재시작한 후 플러그인 메뉴에서 패치 관리라는 이름의 새 플러그인을 사용할 수 있게 됩니다. 이 플러그인을 사용하면 사용자 패치 목록과 상태를 확인하고 패치를 활성화/비활성화할 수 있습니다. 패치 관리자 정보 메뉴 항목을 탭하여 자세한 정보를 확인할 수 있습니다.

패치 이름 붙이기

패치의 실행 순서는 패치 이름에 있는 숫자의 첫 번째 숫자에 따라 결정됩니다. 예: 2-TOC-title-patch.lua 같은 숫자로 시작하는 패치가 여러 개 있는 경우, 실행 순서는 자연스러운 순서인 (1<10<11<12<13<…<109<…<10010)입니다. 이미 만들어진 패치를 다운로드하는 경우 파일 이름을 변경할 필요가 없습니다.

다음으로 시작하는 이름:

  • 0 - 업데이트 후 매우 이른 시간에 한 번만 실행됩니다
  • 1 - KOReader가 시작될 때마다 매우 일찍 실행됩니다(다만 해당되는 경우 0시 이후)
  • 2 - KOReader를 시작할 때마다 UIManager가 준비된 후 실행됩니다
  • 3-7 - 예비
  • 8 - 끝내기 전과 설정이 저장되기 전에 실행됩니다
  • 9 - 끝내기 직전에 실행됩니다

사용자 패치에 문제가 있는 경우 패치 파일을 제거하여 이전 구성으로 되돌릴 수 있습니다.

문서 준비와 전송
독자에게 책 전송하기
KOReader는 PDF와 EPUB, DJVU, MOBI, CBZ, CBT, DOCX, RTF, HTML, TXT, XPS, FB2, PDB, CHM, ZIP 파일을 지원합니다.

KOReader는 PDF 문서를 리플로우하여(이 옵션은 PDF와 같은 고정 레이아웃 문서를 읽는 동안 하단 메뉴에서 찾을 수 있음) 전자 리더의 작은 화면에 적합하게 만들 수 있습니다. 하지만 최상의 결과를 얻으려면 문서를 보다 유연한 EPUB 형식으로 변환하는 것이 좋습니다. Calibre(https://calibre-ebook.com/)를 사용하면 모든 종류의 형식 간 변환을 수행할 수 있습니다.

문서를 내 기기로 전송하는 방법은 여러 가지가 있습니다. 전송된 문서를 색인화하려면 KOReader를 다시 시작해야 할 수도 있습니다(또는 Pocketbook/안드로이드를 사용하는 경우, 디렉터리를 새로 고쳐야 할 수도 있습니다).

내장된 리더 응용 프로그램을 사용하는 것과 동일한 방식으로 파일을 전송하는 것 외에도 기기에 따라 다른 옵션을 사용할 수 있습니다:

  1. KOReader 안에서 USB 대용량 저장 모드: 이 모드는 KOBO나 일부 CERVANTES 기기에서 사용할 수 있습니다.
  2. 클라우드 저장소(Dropbox/FTP/Webdav): 이 기능을 사용하려면 아래 메뉴를 통해 클라우드 저장소 서버의 계정을 추가해야 합니다. 계정을 추가하는 동안 정보 버튼을 통해 진행 과정이 설명됩니다:
     상단 메뉴 (파일 탐색기에서) → → 클라우드 저장소
  3. SSH/SFTP 접속: KOReader는 컴퓨터에서 접속하여 파일을 전송할 수 있는 SSH 서버처럼 작동할 수 있습니다(코보, 킨들, 세르반테스에서 사용 가능). 운영 체제에서 지원하는 경우 SFTP 애플리케이션(예: Filezilla)를 사용하거나 파일 관리자를 통해 접근할 수 있습니다. 방법은 위키를 참조하세요: https://github.com/koreader/koreader/wiki/SSH
  4. Calibre 전송: Calibre 플러그인을 사용하면 Wi-Fi 연결을 통해 Calibre 라이브러리에서 KOReader 기기로 직접 문서를 보낼 수 있습니다. 방법은 위키를 참조하세요: https://github.com/koreader/koreader/wiki/Calibre-wireless-connection
  5. 뉴스 다운로더: 이 플러그인을 사용하면 RSS와 Atom 뉴스 항목을 HTML 파일로 다운로드할 수 있습니다. 방법은 위키를 참조하세요: https://github.com/koreader/koreader/wiki/News-downloader
  6. Wallabag: 이 플러그인을 사용하면 Pocket과 같은 서버 기반 나중에 읽기 서비스인 Wallabag에서 문서를 검색할 수 있습니다. 자세한 내용은 위키 페이지를 참조하세요: https://github.com/koreader/koreader/wiki/Wallabag
모바일 리더 기기에 맞게 EPUB/PDF 문서 최적화하기
모바일 리더 기기에 맞게 문서를 변환하거나 최적화하는 데 가장 널리 사용되는 응용 프로그램은 Calibre(EPUB, Kindle 위주)와 K2pdfopt(PDF 위주)입니다.

Calibre는 다양한 형식으로 문서를 변환할 수 있습니다. 또한 가장 인기 있는 전자책 형식인 EPUB과 Kindle로 전자책을 편집하고 문제를 해결할 수 있는 편집기가 내장되어 있습니다. https://calibre-ebook.com/

K2pdfopt는 모바일 전자책 리더와 스마트폰에 맞게 PDF/DJVU 파일을 최적화합니다. 다단로 구성된 PDF/DJVU 파일에서 잘 작동하며 스캔한 PDF 파일에서도 텍스트를 가변형으로 볼 수 있습니다. 또한 일반적인 PDF 복사/자르기/크기 조정/OCR 조작 도구로도 사용할 수 있습니다. https://www.willus.com/k2pdfopt/

SingleFile은 모질라 파이어폭스와 구글 크롬, 마이크로소프트 엣지용 브라우저 확장 프로그램입니다. 이 확장 프로그램을 사용하면 웹 페이지를 모든 리소스(이미지, 스타일시트, 스크립트)가 포함된 단일 .html 파일로 저장할 수 있습니다. https://github.com/gildas-lormeau/SingleFile#install

ZIP 파일 지원
KOReader는 ZIP 파일로 저장/압축된 파일을 읽을 수 있습니다. ZIP 파일에는 html, txt 또는 이미지와 같은 다양한 유형의 문서가 포함될 수 있습니다. ZIP 압축에 있는 파일 유형에 대한 힌트를 제공하면 KOReader가 ZIP 파일을 올바르게 열 수 있습니다. 파일 확장자만 .zip으로 지정하면 KOReader는 이 압축에 이미지가 포함된 것으로 간주하고 CBZ 파일처럼 열려고 시도합니다.

기본적인 경우: 나의파일이름.zip - 이미지의 zip 파일로 가정합니다(그리고 MuPDF 엔진에서 엽니다)

zip 파일에 다음 유형 중 하나(단순한 이미지 대신)가 포함된 경우, 거기에 맞는 이름을 지어주세요:

  • FictionBook - 나의파일이름.fb2.zip
  • HTML - 나의파일이름.htm.zip / 나의파일이름.html.zip
  • 텍스트 로그 파일 - 나의파일이름.log.zip
  • Markdown - 나의파일이름.md.zip
  • 서식 있는 텍스트 형식 - 나의파일이름.rtf.zip
  • 일반 텍스트 - 나의파일이름.txt.zip

문서 이름이 이렇게 지정되지 않은 경우, 잘못된 리더/포맷 엔진이 사용되었을 가능성이 높으며 이 파일에 대한 리더 엔진 없거나 잘못된 파일 메시지가 표시됩니다. 위에 설명된 대로 이 파일의 이름을 바꾸거나 파일 이름을 길게 누른 다음 다음으로 열기...를 선택하여 올바른 엔진을 선택할 수 있습니다.

모바일 리더 기기에 맞게 만화 최적화하기
전자 잉크 기기로 만화책을 읽는 것을 좋아한다면, 더 나은 경험을 할 수 있는 몇 가지 팁이 있습니다. 만화책이 기기에서 어떻게 보이는지 만족스럽지 않다면, 나만의 만화 파일을 준비하는 것이 가장 좋은 해결책입니다.
파일 유형

MOBI와 EPUB과 같은 파일 유형은 주로 이미지 콘텐츠(예: 만화/코믹)를 처리하는 데 적합하지 않습니다. KOReader의 MOBI 지원은 미미하거나 아예 없으며, EPUB의 텍스트 렌더링 우선 순위로 인해 전체 페이지 이미지를 표시할 때 문제가 발생할 수 있습니다(자세한 기술 세부 사항은 여기를 참조하세요): https://github.com/koreader/koreader/issues/9163#issuecomment-1146637205).

가장 좋은 방법은 KOReader에서 더 잘 작동하는 CBZ 파일을 만드는 것입니다. 확장자가 .cbz인 파일은 압축되지 않은 이미지 모음인 zip 파일입니다. 이 파일은 여러 이미지(일반적으로 .png나 .jpeg 파일)를 코믹 페이지로 구성하여 저장합니다. KOReader는 CBZ 파일과 유사하지만 TAR 압축 형식을 사용하는 CBT 파일도 열 수 있습니다.

만화를 CBZ로 변환하는 가장 쉬운 방법은, 다양한 웹사이트에서 만화를 이미지로 가져온 다음 Kindle 코믹 변환기를 사용하여 변환하는 것입니다./span> (https://github.com/ciromattia/kcc). 두 페이지 분할을 가로로 자동 회전하고 디더링을 적용하여 전자잉크에 렌더링된 그라디언트가 들쭉날쭉한 레이어 없이 부드럽게 디더링되도록 하는 등 몇 가지 유용한 설정이 있습니다.

해상도

전자책 리더 화면 크기에 맞게 이미지 크기를 조정하면 최상의 결과를 얻을 수 있습니다. 기기의 정확한 화면 크기를 대상으로 하는 화면별 CBZ를 사용하면, 읽는 동안 크기가 다시 조정되지 않습니다. KOReader는 읽는 동안 화면 크기에 맞게 이미지 크기를 축소하지만, KOReader가 크기를 조정하기 전에 디더링 패스를 추가하고 이로 인해 품질이 저하되므로 권장하지 않습니다.

하지만 직접 만든 CBZ를 두 개 이상의 장치에서 사용하거나 나중에 보정할 수 있도록 고해상도를 유지하려는 경우, 가장 좋은 방법은 가장 높은 해상도의 장치에 맞게 CBZ를 제작하는 것입니다. 또는 소스와 일치하는 임의의 "높은" 해상도를 선택해도 됩니다. 물론 소스가 더 낮은 해상도가 아니라면 300 PPI 이상(최대 450 또는 600까지)이 좋습니다. 화면 크기가 8인치, 1500×2000이라면 좋은 목표 해상도입니다. 더 큰 이미지의 경우 장치에서 시도해보고 처리할 수 있는지 확인해야 합니다. 특히 메모리가 부족한 구형 킨들은 3000×4000과 같은 크기에서 어려움을 겪을 수 있습니다. 프로세스를 자동화하는 경우 저품질 버전과 고품질 버전(예: 200/400 또는 300/600 DPI)을 모두 만드는 것도 고려해 볼 수 있습니다.

 KOReader는 책의 spine page-progression-direction="rtl" 속성을 구문 분석하지 않습니다. 따라서 파일이 오른쪽에서 왼쪽으로 읽도록 작성되었더라도 페이지 넘김 방향을 변경하지 않습니다. 사용자는 다음을 통해 이를 전환할 수 있습니다:
 상단 메뉴 → → 탭과 제스처 → 페이지 넘기기 → 탭과 스와이프/버튼의 페이지 넘기기 반대로
만화의 고급 준비
다음은 만화 파일 하나를 처리하는 절차의 예시입니다:
  • 다음에서 Kindle 만화 변환기(KCC)를 다운로드하세요 https://github.com/ciromattia/kcc
  • 원하는 소스에서 만화를 다운로드하세요(예: HakuNeko)
  • 만화 파일의 압축을 풉니다. 이제 이미지만 포함된 디렉터리(또는 이미지만 있고 압축은 없는 하위 디렉터리)가 있어야 합니다
  • 이 디렉터리를 KCC로 처리합니다. 각 디렉토리는 단일 CBZ 파일을 생성하므로 한 번에 여러 개의 볼륨을 만들려면 한 번에 여러 디렉토리를 선택하세요

KCC의 몇 가지 유용한 설정:

퍼짐 분할기: 이 기능은 이중 페이지를 감지하여 두 페이지로 회전하거나 잘라낼 수 있습니다.

크기 확대: 목표 해상도보다 작은 이미지를 확대합니다.

KOReader의 몰입형 모드
만화를 더욱 몰입감 있게 즐기고 싶으시다면, CBZ 파일을 만들 때 KCC에서 사용 중인 기기의 화면 해상도와 정확히 동일한 해상도를 선택했는지 확인하세요. 또한 KOReader에서 만화를 읽는 동안 하단의 진행률 표시줄을 숨길 수 있습니다. 그러면 방해 요소 없이 만화를 전체 화면으로 몰입감 있게 볼 수 있습니다.
부드러운 그라데이션을 위한 추가 단계  고급
KCC를 사용하여 CBZ 파일을 만든 후, 이미지에 추가 디더링을 적용할 수 있습니다. 이렇게 하면 전자잉크에 렌더링된 그라데이션이 들쭉날쭉한 레이어 없이 매끄럽게 디더링됩니다. 이 단계는 KOReader에서 더 잘 보이는 미세 조정된 만화 볼륨을 생성합니다.

ImageMagick을 사용하여 이미지를 처리할 수 있습니다. ImageMagick은 래스터 이미지를 표시, 생성, 변환, 수정 및 편집할 수 있는 강력한 오픈 소스 소프트웨어 제품군입니다. ImageMagick을 다운로드하여 명령줄에서 사용하여 한 번에 많은 이미지를 처리할 수 있습니다.

  1. KCC로 생성하여 준비된 CBZ 파일의 압축을 풉니다,
  2. 두 개의 배치 파일(.BAT)과 GIF 색상 템플릿이 포함된 다음 파일(dither_pack.zip)을 다운로드합니다: https://disk.yandex.com/d/glEebjmPYYp7og
  3. 처리할 이미지 옆에 있는 다음 세 개의 파일을 추출하세요
  4. magick_manga_ditherN_resize.bat를 편집하여 해상도를 기기 해상도로 변경하세요(BAT 파일 위치 모두에서).

     다음 단계에서는 디렉토리에 있는 모든 이미지를 처리된 이미지로 바꿉니다. 따라서 원본을 보관하려면 다른 디렉토리에 사본을 만들어 두어야 합니다.

  5. _magick_manga_ditherN_resize.bat를 실행하여 이미지의 크기를 조정하고 디더링하세요.
  6. 7-Zip이 설치되어 있다고 가정하고, _batch_archive.bat을 실행하여 처리된 모든 이미지를 CBZ 압축으로 압축하세요.

["/storage/emulated/0/Books/manga"] = {
["inverse_reading_order"] = true,

본문 들여쓰기 문제

제목, 표제나 구분 기호 또는 예상치 못한 본문 들여쓰기가 발생하면, 문서의 스타일을 epub.css에서 html5.css로 전환해 보세요. 일부 출판사는 웹 브라우저 엔진 기반 리더에서만 책을 읽을 것이라고 가정하고 스타일을 "재설정"하지 않기 때문에 이런 종류의 문제가 발생합니다.

 상단 메뉴 → → 스타일
- 기본 HTML/EPUB(epub.css)은 책 중심의 스타일로, 텍스트 정렬: 오른쪽 맞춤, P로 텍스트 들여쓰기로 설정되어 있습니다. 스타일이 지정되지 않은 책에 더 적합하며, 스타일이 지정된 책의 가독성을 높일 수 있습니다. - 위에서 언급한 것처럼 책에 들여쓰기 문제가 있는 경우 HTML5(html5.css)로 전환해 볼 수 있습니다. 이 스타일은 웹 브라우저에서 렌더링할 때와 같이 텍스트 정렬: 왼쪽, P에 텍스트 들여쓰기 없음, P 사이에 간격 등의 기본 스타일을 제공하지만, 발행인이 스타일을 지정하지 않은 경우에는 보기에 좋지 않습니다.

책 표지 캐시 데이터베이스 정리하기

KOReader는 책 표지를 더 빠르게 표시할 수 있도록 책 표지의 캐시를 보관합니다. 정상적인 상태에서는 이 캐시가 너무 커지지 않습니다. 많은 책을 제거하거나 디렉토리의 이름을 변경하는 경우, 책 표지 캐시 데이터베이스를 정리하여 공간을 확보할 수 있습니다. 이 메뉴에서 정리할 수 있습니다:

 상단 메뉴 (파일 탐색기에서) → → 설정 → 모자이크와 상세 목록 설정 → 책 정보 캐시 관리
파일 캐시 지우기

PDF/DJVU 문서를 읽을 때, KOReader는 마지막으로 읽은 페이지의 캐시를 유지합니다. 따라서 책을 다시 열면 더 빠르게 읽을 수 있습니다. 그러나 오랫동안 KOReader를 사용하면서 많은 PDF/DJVU 문서를 읽으면 이 캐시가 커질 수 있습니다. 이러한 캐시 파일은 KOReader 설치 디렉토리의 캐시 폴더 아래에 보관됩니다. 이 캐시를 지우고 공간을 확보하려면 해당 폴더로 이동하세요:

 상단 메뉴 (파일 탐색기에서) → → 추가 도구 → 개발자 옵션 → 캐시 지우기
 이 공간이 꼭 필요한 경우에만 이 캐시를 지우세요. 공간이 충분하다면 캐시를 지워도 아무런 이점이 없습니다. 대신 리더가 해당 페이지를 다시 렌더링해야 하므로 성능이 저하될 수 있습니다.

문제나 제안 사항에 대하여 연락하는 방법
KOReader는 GitHub를 프로젝트 기반으로 사용합니다. 문제나 제안 사항은 GitHub 페이지에서 보고할 수 있습니다. 무료 GitHub 계정만 있으면 됩니다. 이슈 페이지를 열려면 이 링크를 클릭하세요: https://github.com/koreader/koreader/issues

이슈(논쟁) 페이지에서 새 이슈 버튼을 클릭합니다. 하지만 그 전에 검색창을 사용하여 동일한 주제에 대한 기존 이슈가 이미 있는지 확인하는 것이 좋습니다. 이미 열려 있는 다른 이슈를 확인하여 진행 과정에 대한 아이디어를 얻을 수도 있습니다.

새 이슈 버튼을 클릭하면 두 가지 옵션이 표시됩니다:

해당 항목을 클릭하고 페이지의 안내에 따라 문제나 제안을 신고하세요. 버그를 신고하는 경우 자세한 디버그 로그는 문제를 파악하는 데 매우 유용합니다. 다음에서 활성화할 수 있습니다:

 상단 메뉴 (파일 탐색기에서) → → 추가 도구 → 개발자 옵션 → 디버그 로깅 사용 + 자세한 디버그 로깅 사용
